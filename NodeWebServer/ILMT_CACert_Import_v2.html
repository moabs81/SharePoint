Preface: OpenSSL Toolkit

At the time of this writing, there is not a binary installation of OpenSSL designated for use by SECU. The author has, on both Windows 7 and Windows Server 08R2, used with success the distribution found here which, while not officially supported by the OpenSSL project, is referenced directly by the project as a usable installation for Windows operating systems. The name of the downloaded executable is “Win64OpenSSL-1_0_2L.exe”.

1.) Click next.
2.) Click “I accept the agreement” and click next.
3.) Note the default installation path is at the root of C. This can be changed if desired. Click next when ready to proceed.
4.) The installation will place a program shortcut in the start menu. Click next.
5.) It is your preference as to whether you want to place OpenSSL dll files into the Windows system directory; the author has selected openSSL’s bin directory in order to keep the toolkit as isolated as possible and to avoid any unnecessary changes to the OS’s directories.
6.) Confirm the installation details and click Install.
7.) The installation will proceed.
8.) Uncheck the donation box and click finish. Or don't uncheck it, if you're feeling generous.
9.) Add your installation directory to your Path environment variable.
10.) Close any open instances of cmd and reopen. Test the path by typing openssl which should place you in the openSSL command mode. Type exit to exit this mode.

Congratulations! You have installed openSSL!

Importing Certificate to ILMT

ILMT requires separate certificate chain and key files in .PEM format. The certificate received from the CA must first be imported into the Windows Keystore to be be converted to this format.

1.) Open a command prompt in administrator mode. If necessary, change drives to the drive on which your .pfx keyfile resides by typing pushd 'drive-letter':.
1.) Take the certificate received from the SECU CA (with the extension of .cer or .pfx and import it into the Windows Certificate Store.
Open the Microsoft Management Console from the Start menum.
Add the Certificates snap-in.
Choose the Computer account, the Local computer, and click Finish.
Click OK to finish opening the Certificates snap-in.
3.) Create the key .PEM file using the openSSL command openssl pkcs12 -in certname.pfx -nocerts -out key.pem -nodes where certname.pfx is the name of your .pfx certificate and key, and key.pem is the name of your new key file in the correct format.
